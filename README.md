# Kairos（καιρός）

**AI 智能体的认知记忆系统——合时之忆，恰如其分。**
*Cognitive memory infrastructure for AI agents — the right memory at the right moment.*

> Kairos（καιρός，古希腊语「恰当的时机」）——记忆的价值不在于「记住了什么」，而在于「在正确的时机以正确的方式用到」——并且知道什么时候自己不知道。

## 一、概览 · Overview

Kairos 是一个为 AI 智能体设计的**认知记忆系统**。不同于传统键值存储或向量数据库，它将记忆建模为**多维认知空间**——不仅记录「发生过什么」，还追踪每段记忆的使用模式、见证可信度、时间结构与认知结构占位价值。

系统以**第一性原理正向链**驱动设计：从对记忆本质的认知出发，推导设计理念，约束架构选择，划定系统上限。逆向链则在实践触及天花板时反过来暴露认知的边界。

| 核心设计维度 | 说明 |
|:------------|:------|
| 架构层次 | 七层功能栈（接入/WM/存储/策略/元认知/宪法主权面/推理皮层）+ 两个正交治理面（监督平面、注意力调度器） |
| 认知度量轴 | 五维：使用价值、见证价值、时间、认知完整性、可及性 |
| 设计原则 | P1–P6，自第一性原理推导 |
| 核心新概念 | 注册表、编译器、结构化通信单元、CRI、三区域知识管道、谱系、知识灭绝、MTL、热纠偏 |

## 二、设计哲学 · Design Philosophy

### 第一性原理链

```
认知（是什么）→ 设计理念（为什么）→ 架构（怎么做）→ 上限（天花板在哪）
```

系统的认知基础定稿于 [`cognitive-foundation.md`](docs/foundation/cognitive-foundation.md)（约 900 行），包含五轴度量空间、六机制最小规范集、多重记忆分类、元记忆四维度、P1–P6 原则的认知层论证。

### 核心理念

| 理念 | 含义 | 架构落点 |
|:-----|:------|:---------|
| **辞典式排序** | 身份 > 探索 > 宪法 > 校准 > 认知完整性 > 时间 > 间接度，七级链为宪法级约束 | 策略层辞典式裁决器（§3.2） |
| **价值独立性公理** | 使用权重不能无声改写见证锚定——「好用」≠「真实」 | 双副本分离 + S-14 语境自指禁令 |
| **分域真理观** | 常规任务→实用论、更新合并→融贯论、冲突校准→符合论，三级域三种真理标准 | 操作真理↔规范真理双轨切换协议 |
| **保守倾向** | 平局→NO-OP、不确定→默认保守、跨域回退→规范真理 | 保守倾向闸门（§3.2），与 CRI 协同作用于检索降级 |
| **指针原则** | 引用而非移动数据——注册表、路径空间、双副本分离是同一模式的实例 | 指针通信协议（§0.4）+ 注册表 + MTL |
| **他律性约束** | 外部校准供给上限决定系统认知天花板。三阶段演进：纯他律→受限自主→混合自主 | 外部校准降级状态机（§10.9）|

### 设计原则 P1–P6

六条原则从认知基础推导而来，约束全系统架构决策：

| 原则 | 核心声明 | 架构约束 |
|:-----|:---------|:---------|
| **P1 使用即价值** | 使用是多目标向量，不可聚合力单标量。仅在操作层充当默认驱动源 | 使用价值轴双副本 + 差异检验 |
| **P2 契约主导生命周期** | 常驻/按需/环境/临时四类契约决定激活策略而非存储位置 | 契约运行时映射 + 激活权重分配 |
| **P3 激活-存储解耦** | 契约决定激活策略，存储层统一存放 | 存储层统一 LTM + 路径空间 |
| **P4 遗忘是适应性优化** | 遗忘是资源约束下的优化，潜伏势能缓冲后悔成本 | 二维遗忘曲面遗忘调度器 |
| **P5 探索是认知边界测绘** | 独立于工具性使用排序，预算由元认知层独立分配 | 探索预算 + 盲区标注路由 |
| **P6 禁止无声丢失维度信息** | 任何标量化操作不可永久丢失某维度的信息，丢失须可审计回溯 | P6 受控例外 + 序数压制幅度记录 |

### 关键张力声明

- **他律性悖论**：系统的认知上限最终由外部校准供给决定，但外部校准的充分性本身不可被系统自我证明。此悖论不试图消解，而是通过弱自反性声明和审计庭独立性来管理。
- **使用价值 vs 见证价值**：使用权重（好用）与见证锚定（真实/自洽）之间结构性冲突，非默认和谐。差异检验是两者之间的仲裁机制。
- **探索 vs 保守**：探索扩展认知边界，保守防止不可逆错误。两者是正交决策变量——探索决定方向，保守决定幅度。

### 七层功能栈 + 正交组件

```
┌──────────────────────────────────────────────────────────────────┐
│  ① 宪法主权面 (§1)  S-01~S-20  20条安全红线                      │
│  ├─ 外部校准端口 · 宪法修订端口 · 强制冻结权限                     │
│  └─ 宪法解释层：抽象原则→语境判例映射                              │
├──────────────────────────────────────────────────────────────────┤
│  ② 元认知层 (§2)  检测器族 + 治理器族 + 自观察记忆                  │
│  ├─ CRI 监测 · 热纠偏触发 · 分布偏移检测 · 盲区标注                 │
│  └─ 种子生命周期追踪 · 元记忆偏置监测 · 叙事连贯性检测              │
├──────────────────────────────────────────────────────────────────┤
│  ③ 策略层 (§3)  价值上下文管理器 + 辞典式裁决器 + 预测器+调节器     │
│  ├─ 七级辞典式裁决（身份>探索>宪法>校准>完整性>时间>间接度）         │
│  ├─ 检索深度分级 L0 浅层/L1 中层/L2 全面 ← CRI 联动降级             │
│  └─ 决策模式 L0 全自动→L3 人工确认 ← CRI/偏置告警联动               │
├──────────────────────────────────────────────────────────────────┤
│  ④ 推理皮层 (§4)  汇聚式多路径融合 + 见证锚定叙事自洽度评估         │
├──────────────────────────────────────────────────────────────────┤
│  ⑤ 存储层 (§5)  双副本分离 + 三区域知识管道 + 遗忘调度器            │
│  ├─ 加工区→验证区→正式库（三级门闸 + 退回回流闭环）                  │
│  ├─ 注册表（确定性状态，五根键域 O(h) 路径寻址）                   │
│  ├─ 解决方案谱系 · 多级蒸馏 · 知识灭绝/化石 · MTL 地址映射         │
│  └─ 关系索引 · 6 维混合检索 · 情绪 VAD 维度 · 升华管道              │
├──────────────────────────────────────────────────────────────────┤
│  ⑥ 工作记忆层 (§6)  WM 槽位 + 整合窗 + 多路径结果集                │
│  └─ 沙箱验证环 · 策略提炼旁路 · 模拟隔离区                         │
├──────────────────────────────────────────────────────────────────┤
│  ⑦ 接入层 (§7)  多源摄取 + 编译器 + 记忆 Provider                  │
│  └─ 编译器(L1 规则+L2 模型) → 结构化通信单元（跨层通信标准格式）    │
├──────────────────────────────────────────────────────────────────┤
│  ⑧ 注意力调度器 (§9，横切)  注意力预算分配 + 容量约束 + 动态调权    │
├──────────────────────────────────────────────────────────────────┤
│  ⑨ 监督平面 (§1.7，正交)  审计庭 → 仅对外部物理信令负责            │
└──────────────────────────────────────────────────────────────────┘
```

### 核心技术概念

| 概念 | 认知层定位 | 架构实现 |
|:-----|:-----------|:---------|
| **注册表** | 确定性状态——与概率性记忆互补，回答「我当前是什么」 | 树形层级键值对，五根键域（自身身份/用户/环境/会话/云端），O(h) 路径寻址 |
| **编译器** | 意图净化→符号映射→结构化生成，是自然语言与系统内部的隔离边界 | 接入层子模块，L1 规则 + L2 模型两级，输出统一的结构化通信单元 |
| **结构化通信单元** | 跨层通信的标准格式——自然语言仅在对外接口中存在 | Schema：unit_id/source/target/op/symbols/payload/constraints/metadata |
| **上下文腐烂(CRI)** | 随交互轮次累积，有效信息利用率系统性下降。三阶段：噪声锚定→聚集→排斥 | [0,1] 综合指数（注意力熵+有效信息利用率+任务成功率），>0.6 触发检索深度降级 |
| **三区域知识管道** | 原始→验证→归档，三级生产阶段 | 加工区/验证区/正式库，三级门闸 + 退回回流，`memories.hall` 字段标记 |
| **解决方案谱系** | 同一原始素材的多条语境化表征分支 | `solution_branches` 表，四规则：分支创建/共享锚定/融合/休眠 |
| **多级蒸馏** | 碎片→会话→日总结→体系→元规则 | `memories.distill_level` 0-4 字段标记，非逐级递进 |
| **知识灭绝/化石** | 环境变迁驱动的知知识失效，非价值遗忘 | 化石区拓扑保留+内容摘要化，元数据可反向激活恢复 |
| **MTL 地址映射** | 逻辑地址与物理存储分离 | LMA→PMA 二级映射，TLB 缓存高频映射，冷迁移透明 |
| **热纠偏** | 使用权重反周期校正——防止注意力自激 | 权重分布熵监测器 + 加速衰减注入辞典式裁决器 |
| **检索深度分级** | L0 浅层/路径匹配 / L1 中层/摘要匹配 / L2 全面/全量重建 | CRI 联动：>0.6 强制 L0，>0.4 降 L1 |
| **决策模式分级** | L0 全自动 / L1 轻量提示 / L2 多方案推荐 / L3 人工确认 | 价值上下文管理器按 CRI+偏置告警动态选择 |

### 安全体系

20 条安全红线（S-01~S-20），涵盖六大领域：

| 领域 | 红线示例 |
|:-----|:---------|
| 身份与机密 | S-01 身份注册表被篡改、S-06 路径逃逸、S-07 外部源注入 |
| 访问控制 | S-13 注册表权限越限、S-19 注册表根键域权限 |
| 数据完整性 | S-02 使用权重篡改见证锚定、S-08 双副本不一致、S-15 记忆投毒 |
| 审计独立 | S-04 审计庭被控制、S-18 审计独立 |
| 系统边界 | S-09 编译器投毒、S-16 恶意跨层调用 |
| 架构不变量 | S-10 遗忘调度器被禁用、S-14 语境自指禁令 |

> 详细定义见 [`docs/security/security-specification.md`](docs/security/security-specification.md)
