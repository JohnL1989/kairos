# Kairos 性能基准计划

> **定位**：定义性能测试方法、数据规模和硬件基线。NFR 定义了「目标值」，本文定义「如何测出目标值」。

---

## 一、测试环境基线

| 资源 | 标准模式 | 轻量模式 |
|:----|:--------:|:--------:|
| CPU | 4 核（x86_64） | 2 核 |
| 内存 | 8 GB | 2 GB |
| 磁盘 | SSD, 50 GB 可用 | SSD, 10 GB 可用 |
| OS | Ubuntu 22.04 / Windows 11 | 同左 |
| 数据库 | PostgreSQL 15 | SQLite 3.40 |

## 二、测试数据规模

| 测试级别 | 记忆数 | 说明 |
|:--------|:------:|:-----|
| 小规模 | 1,000 | 功能验证 |
| 中规模 | 50,000 | 典型使用场景 |
| 大规模 | 500,000 | 压力测试 |
| 极限 | 1,000,000 | 容量边界验证 |

## 三、测试方法

### 3.1 写入延迟

> **通过/失败判据**：P50 ≤ 50ms, P95 ≤ 200ms（NFR 规格 §一）

```
1. 预热：写入 1000 条记忆
2. 测量：逐条记录写入延迟（连续写入 10000 条）
3. 统计：P50/P95/P99
4. 变量：内容长度（256B/1KB/8KB）、嵌入维度、索引类型
```

### 3.2 检索延迟

```
1. 准备：写入 N 条记忆（N=1000/50000/500000）
2. 测量：随机选取 1000 条已知路径进行路径前缀检索
3. 测量：随机选取 1000 条记忆进行语义检索
4. 统计：P50/P95/P99
5. 变量：存储规模、向量索引类型（IVFFlat/HNSW）、probes 参数
```

### 3.3 吞吐量

```
1. 准备：写入 50000 条记忆
2. 写入吞吐：并发 10/50/100 个客户端，持续 60s
3. 检索吞吐：并发 10/50/100 个客户端，持续 60s
4. 混合负载：70% 检索 + 30% 写入，持续 120s
5. 统计：平均吞吐量 + P95 延迟
```

### 3.4 升华批处理

```
1. 准备：选中 100 条记忆进入升华队列
2. 触发：强制触发升华（跳过空闲检测）
3. 计时：从 raw 阶段开始到 behavior 阶段产出
4. 统计：平均完成时间 + 各阶段耗时分布
```

### 3.5 磁盘增长

```
1. 初始：记录空库磁盘占用
2. 写入：每写 10000 条记录一次磁盘占用
3. 触发升华：记录升华后的磁盘变化
4. 触发遗忘：记录遗忘归档后的磁盘释放
5. 绘制：写入量 vs 磁盘占用曲线
```

## 四、性能回归阈值

| 指标 | 回归阈值 | 说明 |
|:----|:--------:|:-----|
| 写入延迟 P50 | 超过基线 50% | 触发告警 |
| 检索延迟 P50 | 超过基线 100% | 阻塞发布 |
| 吞吐量 | 低于基线 30% | 触发告警 |
| 内存泄漏 | 连续 24h 增速 > 1%/h | 阻塞发布 |
| 磁盘增速 | 超过预期 100% | 触发告警 |
