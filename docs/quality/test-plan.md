# Kairos 测试计划

> **定位**：v1.0 核心路径与边界用例的具体定义。与 `test-strategy.md`（方法论/金字塔）互补——策略说"怎么测"，本文件说"测什么"。
>
> **代码启动前**应完成核心路径用例定义（TC-* 的输入/预期输出/边界条件），按 TDD 或 test-first 方式驱动实现。具体用例数待代码启动后根据实际接口数量确定。

---

## §1 测试范围

### v1.0 核心覆盖

| 层级 | 覆盖目标 | 用例数 |
|:----|:--------|:-----:|
| 单元 | 各组件核心函数（存储/WM/策略/宪法主权面） | 待定（代码启动后根据实现接口数量确定） |
| 集成 | 组件间接口（WM↔存储、PM↔存储、主权面↔PM） | 待定（代码启动后根据组件数量确定） |
| E2E | 6 条关键路径（写入→检索→遗忘→升华→校准→审计） | 6 |
| 安全 | 19 条安全红线逐条验证 | 19 |

### 不覆盖

- UI 测试（无图形界面）
- 性能基准（由 benchmark-plan.md 覆盖）
- v1.1+ 功能（定向遗忘/社会性校准等）

---

## §2 进入/退出准则

| 准则 | 条件 |
|:----|:-----|
| **进入**（何时开始测） | 核心组件接口稳定、DB schema 已迁移 |
| **退出**（何时算测完） | 单元覆盖率 ≥80%、6 条 E2E 全通过、19 条安全红线逐条验证通过 |
| **暂停** | 阻塞性 P0 bug 未修复时暂停 E2E，修完继续 |

## §3 核心路径用例（代码启动前）

### 3.1 写入（W-01 ~ W-07）

| ID | 前置 | 步骤 | 输入 | 预期输出 |
|:---|:-----|:----|:----|:--------|
| TC-W01-001 | 系统运行 | `kairos write kairos://test/greeting "hello"` | 路径+内容 | 返回路径 `kairos://test/greeting` |
| TC-W01-002 | 已有一条 W-001 | 路径前缀检索 `kairos://test/` | 检索参数 | 返回包含该记忆的列表 |
| TC-W01-003 | 无 Key 状态（运行时） | `curl -X POST /v1/memories` | 无 Authorization | 401 | 启动级无 Key 为连接被拒绝（S-01 启动拦截）；运行时缺 Key 返回 401
| TC-W02-001 | 系统运行 | 指定 `contract=permanent` 写入 | 常驻契约 | 遗忘调度器扫描 10 轮后该记忆仍存在 |
| TC-W04-001 | 系统运行 | 写入时附带 VAD `{"v":0.5,"a":0.3,"d":0.8}` | VAD 元数据 | 检索结果中包含 VAD 字段 |
| TC-W07-001 | 系统运行 | 写入含敏感模式内容（如 API Key） | 敏感内容 | DB 中该字段为密文 |

### 3.2 路径检索（R-01）

| ID | 前置 | 步骤 | 输入 | 预期输出 |
|:---|:-----|:----|:----|:--------|
| TC-R01-001 | 路径下有 10 条记忆 | `kairos search --path kairos://test/` | 路径前缀 | 返回该路径下全部 10 条 |
| TC-R01-002 | 路径不存在 | `kairos search --path kairos://nonexistent/` | 不存在路径 | 返回空列表，非 404 |
| TC-R01-003 | 路径为空 | `kairos tree` | 空路径 | 显示空目录树 |

### 3.3 语义检索（R-02）

| ID | 前置 | 步骤 | 输入 | 预期输出 |
|:---|:-----|:----|:----|:--------|
| TC-R02-001 | 写入 3 条相关记忆 | `kairos search "query text"` | 查询文本 | 返回按相似度排序的 Top-K |
| TC-R02-002 | 无匹配 | 检索完全不相关内容 | 无关查询 | 空列表（或最低相似度项） |

### 3.4 遗忘与复兴（F-01 ~ F-03）

| ID | 前置 | 步骤 | 输入 | 预期输出 |
|:---|:-----|:----|:----|:--------|
| TC-F01-001 | 写入 1 条临时契约记忆 | 等待遗忘调度器运行 N 周期 | — | 被标记为遗忘候选 |
| TC-F02-001 | 1 条低使用记忆 | 手动触发潜伏重估 | 重估信号 | 记忆被重估，保留或归档 |
| TC-F03-001 | 已遗忘记忆 | 通过关联记忆检索触发复兴 | 关联检索 | 记忆复兴，回归可用状态 |

### 3.5 校准与降级（C-01 ~ C-04）

| ID | 前置 | 步骤 | 输入 | 预期输出 |
|:---|:-----|:----|:----|:--------|
| TC-C01-001 | 系统运行 | `POST /v1/calibrate` | 校准信号 | 见证锚定更新 |
| TC-C03-001 | 系统运行 | `kairos admin freeze` | 冻结命令 | 内部环暂停，冻结状态查询返回 frozen |
| TC-C04-001 | 系统运行 | `kairos admin degrade safety-sleep` | 降级命令 | 系统进入安全休眠态，仅保留基本存取 |

### 3.6 E2E 关键路径（6 条）

| ID | 路径 | 用例 |
|:---|:-----|:----|
| E2E-01 | 写入→检索 | 写入 3 条记忆，路径检索和语义检索均确认存在 |
| E2E-02 | 写入→遗忘→复兴 | 写入→等待遗忘→确认遗忘→关联检索触发复兴→确认复兴 |
| E2E-03 | 写入→升华 | 写入→空闲触发升华→查询升华状态→确认 raw→item 升格 |
| E2E-04 | 校准→见证更新 | 注入外部校准信号→确认见证锚定更新→检索含新锚定值 |
| E2E-05 | 冻结→恢复 | 冻结系统→确认不可写→解冻→确认可写 |
| E2E-06 | 审计链验证 | 写入→修改→删除→查询审计日志→验证 HMAC 链完整性 |

### 3.7 安全红线（19 条逐条验证）

参见 `quality/test-strategy.md §2.2` 安全红线验收表（19 条逐条验证方法）。

## §4 测试数据准备

### 种子数据

```python
# test_data/seeds.py — 测试启动时自动加载
seeds = [
    {"path": "kairos://test/core/greeting", "content": "hello world", "contract": "permanent"},
    {"path": "kairos://test/core/farewell", "content": "goodbye", "contract": "ondemand"},
    {"path": "kairos://test/domain/topic_a", "content": "AI memory systems", "contract": "environmental", "vad": {"v":0.6,"a":0.4,"d":0.7}},
    {"path": "kairos://test/domain/topic_b", "content": "spaced repetition", "contract": "temporary"},
]
```

### 夹具

- `MockStorageBackend` — 内存存储，测试后自动清理
- `FrozenClock` — 固定时间戳，确保遗忘曲线可预测
- `NullEmbedding` — 返回固定向量，消除 Embedding 模型引入的不确定性

## §5 缺陷跟踪

- **追踪工具**：GitHub Issues（或本地 TODO 文件）
- **标签体系**：`bug/p0`（阻塞）、`bug/p1`（严重）、`bug/p2`（轻微）、`test-missing`（缺用例）
- **流转**：发现 → 开 Issue（含测试用例/日志/环境） → 修复 → 对应测试通过 → 关闭
- **单人简化**：你自己就是 PM/开发/测试，发现即修，修完即关
