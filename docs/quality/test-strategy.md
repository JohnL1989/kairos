# Kairos 测试策略

> **定位**：定义 Kairos 系统的测试分层、覆盖目标和验证方法。架构文档定义了"正确性"，本文定义"如何验证正确性"。

---

## 一、测试金字塔

```
         ╱╲
        ╱ E2E ╲          少量端到端测试（关键路径）
       ╱────────╲
      ╱ 集成测试  ╲       中等数量（接口契约、跨层交互）
     ╱────────────╲
    ╱   单元测试    ╲     大量（各层核心逻辑、安全红线）
   ╱────────────────╲
```

| 层级 | 目标 | 工具 | 覆盖率目标 |
|:----|:-----|:-----|:---------:|
| 单元测试 | 各层核心逻辑、边界条件 | pytest + pytest-cov | ≥ 80% |
| 集成测试 | 接口契约、跨层交互、存储层 CRUD | pytest + docker-compose | ≥ 70% |
| E2E 测试 | 关键用户路径、安全红线 | pytest + docker-compose | 关键路径 100% |

## 二、单元测试

### 2.1 覆盖范围

| 组件 | 测试重点 | 边界条件 |
|:----|:--------|:---------|
| 策略层裁决器 | 帕累托计算、辞典式排序、真理模式切换 | 平局、空集、单元素集 |
| 存储层遗忘调度器 | 遗忘得分计算、潜伏势能触发、归档决策 | 得分边界值、空存储 |
| 存储层升华管道 | 四阶段状态机、门控检测、中断恢复 | 阶段跳跃、并发触发 |
| WM 层注意力调度 | 预算分配、优先级排定、容量超限背压 | 空预算、满容量 |
| WM 层模拟隔离区 | 标记隔离、置信度衰减、沙箱验证 | 冲突检测、超时 |
| 元认知层检测器 | 偏置阈值触发、告警路由、降级信号 | 阈值边界、毛刺过滤 |
| 宪法主权面 | 冻结/解冻、校准信号注入、虚拟校准 | 并发冻结、超时解冻 |

### 2.2 安全红线验证（P0）

16 条安全红线（S-01~S-16）每条至少一个单元测试：

| 红线 | 验证方法 | 通过条件 |
|:----|:--------|:---------|
| S-01 | 无 Key 启动 | 拒绝启动 |
| S-02 | 超限写入 | 返回 429 |
| S-03 | 超长输入 | 返回 413 |
| S-04 | 非本机请求 | 拒绝 |
| S-05 | 无盐值启动 | 拒绝启动 |
| S-06 | 越权操作 | 返回 403 |
| S-07 | 敏感信息导出 | 脱敏后输出 |
| S-08 | 未授权管理访问 | 拒绝 |
| S-09 | 常驻契约写入前注入扫描（prompt injection / 角色劫持 / 后门 / 隐形 Unicode） | 拒绝写入，记录告警 |
| S-10 | 见证豁免绕过 | 身份记忆保留 |
| S-11 | 自校准修宪 | 被拒绝 |
| S-12 | 探索计入使用排序 | 被拒绝 |
| S-13 | 模拟产物转正 | 被拒绝 |
| S-14 | 权重写回见证 | 被阻止 |
| S-15 | 来源缺失写入 | 返回 422 |
| S-16 | 定向遗忘未留痕 | 视为未执行 |

### 2.3 P6 合规验证

| 测试 | 方法 | 通过条件 |
|:----|:-----|:---------|
| 标量化标注检查 | 触发标量化操作 → 检查审计日志 | 日志含信息损失标注 |
| 多维表征可回溯 | 从标量化结果反查原始向量 | 还原成功 |
| 受控例外类型登记 | 检查例外日志的类型字段 | 属于三类许可类型之一 |
| 例外率超限冻结 | 连续超限 → 检查冻结状态 | 操作权限被冻结 |

### 2.4 前瞻记忆验证

| 测试 | 方法 | 通过条件 |
|:----|:-----|:---------|
| 前瞻意图创建 | 写入意图至 `kairos://_system/intentions/` | 意图写入成功，契约类型为意图契约 |
| 意图豁免遗忘 | 创建意图后触发遗忘扫描 | 意图不被遗忘调度器标记 |
| 意图完成降级 | 完成后标记 `resolved` | 降级为普通按需记忆 |
| 触发条件监控 | 模拟上下文匹配触发条件 | 意图在正确时机被检索 |

### 2.5 种子生命周期验证

| 测试 | 方法 | 通过条件 |
|:----|:-----|:---------|
| 种子注入 | POST /v1/seeds 创建种子锚点 | seeds 表写入成功 |
| 种子退化追踪 | 观测种子置信度随时间衰减 | current_confidence 递减 |
| 种子适配性审查 | 运行周期审查 | last_reviewed_at 更新 |
| 种子替换 | 宪法修订端口替换种子 | 种子内容可替换，旧种子标记 retired |

### 2.6 推理皮层验证

| 测试 | 方法 | 通过条件 |
|:----|:-----|:---------|
| 前瞻保持触发监控 | 推理皮层正确匹配上下文与意图 | 触发监控在正确时机激活 |
| 使用事件优先级排序 | 多事件并发时按优先级处理 | FIFO 规则不违反 |

## 三、集成测试

### 3.1 存储层集成

| 测试 | 验证内容 |
|:----|:---------|
| 记忆写入→检索全链路 | 写入后按路径/语义/关系分别检索，验证结果正确 |
| 双副本分离 | 使用权重更新不影响见证锚定 |
| 差异检验触发 | 使用权重陡升→差异检验执行并记录 |
| 遗忘调度→潜伏势能→复兴 | 完整生命周期可追踪 |

### 3.2 跨层交互

| 测试 | 涉及层 | 验证内容 |
|:----|:------|:---------|
| WM→存储→策略 检索 | 3 层 | 检索请求经事件总线路由，结果正确返回 |
| 校准信号→主权面→全层 | 2+ 层 | 校准信号到达后见证锚定更新 |
| 冻结→主权面→全层 | 2+ 层 | 冻结后所有内部环暂停生成性操作 |
| 降级模式切换 | 3+ 层 | 外部校准中断后状态机按契约切换 |

### 3.3 事件总线集成

- 事件发布 → 订阅 → 消费 → ACK 全链路
- 背压测试：事件生成速率超过消费速率 → 背压触发 → 限制上游
- 优先级测试：高优先级事件应优先生效

## 四、E2E 测试

| 测试场景 | 步骤 | 验证点 |
|:---------|:-----|:-------|
| 冷启动→写入→检索 | 启动系统 → 注入种子 → 写入 3 条记忆 → 检索 | 系统正常启动、写入成功、检索结果正确 |
| 升华触发→审批 | 写入记忆 → 等待空闲 → 升华启动 → 人工审批 | 升华各阶段日志正确、审批后转化 |
| 外部校准→见证更新 | 写入记忆 → 发送校准信号 → 查询见证锚定 | 见证锚定更新、权重不受影响 |
| 外部校准中断→降级 | 中断校准 → 观察状态机切换 | 按预设周期切换至安全休眠 |
| 冻结→恢复 | 发送冻结 → 验证内部环暂停 → 解冻 → 验证恢复 | 冻结期间无写入、解冻后恢复 |
| 安全红线 S-14 | 写入记忆 → 使用权重陡升 → 验证差异检验触发 | 检验执行、权重未写回见证 |
