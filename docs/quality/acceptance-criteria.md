# Kairos 验收标准

> **定位**：定义 D-201~D-206 六项代码债务的可判定完成条件，以及 v1.0 发布的硬性检查项。

---

## 一、Code Debt 完成条件

### D-201 逻辑-因果轴完整落地

**验收条件**（全部满足即视为完成）：
1. 因果链完整性检查器：输入一条记忆 ID，可输出该记忆所属因果链的完整节点列表
2. 执行流断裂检测器：程序性记忆的执行步骤序列出现断点时，标记 `execution_flow_broken`
3. 节律记忆时间索引：周期性事件的后续触发预测与实际情况的偏差 ≤ 20%
4. 时间贴现加权：检索排序中物理时间和逻辑-因果时间的加权公式按配置执行

### D-202 动态身份增强

**验收条件**：
1. 身份注册表定期（每 N 周期）适配性审查执行成功
2. 核心/外围区分生效：外围身份可降级（is_identity→false），核心身份不可降级
3. 身份记忆降级门槛由「宪法修订端口」降至「元认知层+宪法解释层联合审批」
4. 所有身份调整事件写入审计日志

### D-203 能力转化状态机

**验收条件**：
1. procedural::nascent → consolidated → degraded 状态机可被代码追踪
2. 状态转换触发条件（验证计数 + 冷却周期）按配置执行
3. 状态转换事件写入使用事件总线
4. 长期不用的程序记忆可退回至 `degraded` 状态（需要 WM 检索才能重新执行）

### D-204 定向遗忘机制

**验收条件**：
1. 通过 CLI/REST API 执行定向遗忘 → 标记路径不可检索
2. 定向遗忘的数据仍保留在存储层（可恢复）
3. 所有定向遗忘操作写入审计日志（含操作者/时间/原因/HMAC）
4. 通过宪法解释层的定向遗忘审查器授权（非自授权）
5. 定向遗忘不逃逸 S-10（不被遗忘的见证仍豁免）

### D-205 外部校准源充分性建模

**验收条件**：
1. 校准源可靠性评估函数实现（输入：校准信号时间序列 → 输出：可靠性分数）
2. 充分性判据函数实现（当前覆盖维度 vs 所需覆盖维度的差距量化）
3. 充分性不足时发出主动告警（而非仅被动降级）
4. 单 Provider 降级按加权规则执行

### D-206 四轴完整度量函数

**验收条件**：
1. 反例覆盖度监测器：跟踪已覆盖推理空间边界 / 总边界
2. 路径禁区标注密度监测器：已标注死胡同路径 / 总探索路径
3. 组合约束网络连通性监测器：约束间可达路径数 / 最大可能数
4. 三项指标可通过 API 查询

## 二、v1.0.0 发布检查项

### 功能检查

| 检查项 | 条件 | 验证方法 |
|:------|:----|:---------|
| 记忆 CRUD | 写入/读取/更新/删除全链路可运行 | E2E 测试通过 |
| 路径空间 | 路径前缀检索和树状浏览正常 | 集成测试 |
| 语义检索 | 向量搜索返回语义相似的记忆 | 精度验证 |
| 双副本分离 | 使用权重写回见证锚定被拒绝（S-14） | 单元测试 |
| 四类契约 | 每类契约的激活/遗忘行为符合定义 | 单元测试 |
| 遗忘调度 | 定时扫描 + 遗忘决策执行 | 集成测试 |
| 升华管道 | raw→item→strategy→behavior 四阶段可追踪 | E2E 测试 |
| 外部校准 | 校准信号接收 + 见证锚定更新 | E2E 测试 |
| 降级模式 | 三模式触发/切换/退出正确 | E2E 测试 |

### 安全红线检查

| 检查项 | 目标 |
|:------|:----|
| 安全红线 S-01~S-16 | 16/16 单元测试通过 |
| 审计链 HMAC 完整性 | 连续审计日志的 HMAC 链可验证 |
| API Key 认证 | read/write/admin 三级鉴权正确 |

### 非功能检查

| 检查项 | 目标 | 验证方法 |
|:------|:----|:---------|

| 检索延迟（路径前缀） | P50 ≤ 20ms, P95 ≤ 80ms | benchmark-plan 3.2 检索延迟测试 |
| 检索延迟（语义） | P50 ≤ 100ms, P95 ≤ 300ms | benchmark-plan 3.2 检索延迟测试 |
| 写入吞吐 | ≥ 100 ops/s | benchmark-plan 3.3 吞吐测试 |
| 检索吞吐 | ≥ 500 ops/s | benchmark-plan 3.3 吞吐测试 |
| 安全红线 | S-01~S-16 全部单元测试通过 | test-strategy 2.2 安全红线验收表 |
