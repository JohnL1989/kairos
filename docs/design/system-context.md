> **定位**：显式描述 Kairos 系统的外部边界——系统负责什么、不负责什么、假设什么。架构文档内部定义了组件职责，本文定义了系统与外部的契约。

## 二、系统边界声明

### 2.1 Kairos 负责什么

| 领域 | 详细 |
|:----|:-----|
| 记忆存储 | 持久化记忆内容、元数据、关系、嵌入向量 |
| 记忆检索 | 按路径/语义/关系/情感/时间多种维度检索 |
| 遗忘管理 | 基于使用价值的遗忘决策 + 潜伏势能保留 |
| 升华提纯 | 空闲时将经验逐步提纯为可复用的行为规则 |
| 价值裁决 | 使用权重与见证锚定的双轴评估与冲突仲裁 |
| 外部校准 | 接收外部校准信号并更新见证锚定 |
| 安全治理 | 16 条安全红线的强制执行 + 审计日志 |
| 系统自省 | 元认知监测、偏置检测、证伪响应 |

### 2.2 Kairos 不负责什么

| 领域 | 说明 |
|:----|:-----|
| 应用层推理 | 不提供通用规划、决策、自然语言理解（推理皮层仅用于记忆相关的最小推理） |
| 对话管理 | 不管理对话状态、会话上下文（可存储但不由 Kairos 维护） |
| 外部编排 | 不调度外部 Agent、服务或工作流 |
| 用户界面 | 不提供面向终端用户的图形界面（提供 API/CLI/Tool） |
| 应用层缓存 | 不替代应用层的业务缓存 |
| 消息队列 | 不替代专用消息队列（事件总线仅用于内部跨层通信） |

### 2.3 Kairos 假设什么

| 假设 | 说明 |
|:----|:-----|
| 外部校准源存在 | 见证价值轴的符合论依赖外部校准信号的定期注入 |
| 存储后端可用 | 依赖 PostgreSQL（标准模式）或 SQLite（轻量模式） |
| 嵌入服务可用 | 需要 LLM Provider 或本地嵌入模型提供向量化能力 |
| 宿主 Agent 有 API Key | 所有外部交互需经过 API Key 鉴权 |
| 单进程部署 | 当前架构假设 Kairos 为单进程服务（多进程需外部负载均衡） |

## 三、外部依赖

| 依赖 | 强度 | 替代方案 |
|:----|:----:|:---------|
| PostgreSQL 15+ | 标准模式必需 | 轻量模式 SQLite |
| pgvector 0.5+ | 标准模式必需 | 轻量模式 sqlite-vec |
| 嵌入模型 API | 标准模式必需 | 本地 BGE-M3（配置可切） |
| LLM Provider API | 升华必需 | 升华可禁用（`--no-sublimation`） |
| 外部校准信号 | 治理必需 | 虚拟校准（受限模式） |
