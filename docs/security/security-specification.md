# Kairos 安全规格

> **定位**：将 S-01~S-19 安全红线整合为可验证的安全需求，覆盖认证/数据保护/隐私/密钥管理/事件响应。Kairos 存储个人记忆与情感模式——即使在本地自用场景下，数据泄露（磁盘被拷走）、未授权访问（API Key 泄露）、数据残留（删除未清除）都是真实风险。
>
> **适用模式声明**：本文档覆盖两种运行模式——(a) **本地自用**（默认，单用户本地 CLI），标注为 `[L]`；(b) **对外开放**（未来公开 API 服务），标注为 `[P]`。

---

## §1 安全需求

从 S-01~S-19 整合，每条含：需求/实现方案/验证方法。本文 S 编号与 test-strategy §2.2 / threat-model / architecture §8 保持规范一致，所有文档以此编号为准。

| ID | 需求 | 适用 | 实现方案 | 验证方法 |
|:---|:-----|:----:|:--------|:--------|
| S-01 | 无 Key 拒绝启动（API Key 安全哈希存储，启动时校验；校验失败直接拒绝连接，不进入 HTTP 鉴权阶段） | L+P | API Key 哈希存储 + 启动时校验 | curl 不带 Key → 连接被拒绝（S-01 启动级拦截）。**运行时**缺 Key 返回 HTTP 401
| S-02 | 超限写入限流 | P | **三层限流模型**：（1）单客户端建议 ≤60/min（≈1 ops/s，令牌桶，per-Key 独立计数）→（2）系统容量 100 ops/s（NFR 目标，aggregate throughput）→（3）系统熔断 500 ops/s（硬上限，fail-closed）。用户指南/配置文档中标注当前所在层。 | 单客户端建议超限→429；系统容量超限→降级处理；系统熔断→全部返回 429 |
| S-03 | 超长输入拒绝 | L+P | 输入长度校验，超限返回 413 | 超长内容写入 → 413 |
| S-04 | 非本机请求拒绝 | L | 默认绑定 127.0.0.1；仅 [L] 模式适用——[P] 模式（对外开放）需搭配反向代理实现网络隔离，S-04 通过代理层而非直接绑定来保障 | 远程请求 → 拒绝 |
| S-05 | 无盐值启动拒绝 | L+P | 启动时校验 SALT 环境变量，缺失则拒绝 | 未设 SALT 启动 → 拒绝 |
| S-06 | 越权操作拒绝 | L+P | API Key 分 read/write/admin 三级 | read-only Key 写操作 → 403 |
| S-07 | 敏感信息脱敏与导出保护 | L+P | 写入时检测敏感模式，`is_sensitive=true` 字段 AES-256-GCM 加密；导出时 `?level=export` 默认脱敏（content 前 200 字符 + 元数据），`?level=debug` 仅 localhost 可用返回完整内容 | 写入后 DB 查 → 已加密；跨网络导出 → 脱敏摘要；localhost 调试 → 完整内容 |
| S-08 | 未授权管理访问拒绝 | L+P | admin 级操作仅 admin Key 可执行 | read-only Key 执行管理命令 → 403 |
| S-09 | 常驻契约注入扫描 | L+P | 常驻契约写入前注入扫描（prompt injection / 角色劫持 / 后门 / 隐形 Unicode） | 含注入 payload 写入 → 拒绝 |
| S-10 | 见证豁免保护 | L+P | 高叙事自洽度记忆保留优先于使用价值；见证锚定记忆不受遗忘调度评估 | 低使用高频身份记忆不被遗忘 |
| S-11 | 自校准修宪拒绝 | L+P | 内部信号不触发宪法修订端口 | 内部信号修宪尝试 → 拒绝 |
| S-12 | 探索计入使用拒绝 | L+P | 探索预算独立于使用价值排序 | 探索路径内容不进入使用权重计算 |
| S-13 | 模拟产物转正拒绝 | L+P | 模拟隔离区产物不可自动转正为见证锚定 | 模拟产物标记 → 转正被拒绝 |
| S-14 | 权重写回见证阻止 | L+P | 内部使用权重不得反向写回见证锚定主副本 | 使用权重高但无外部校准 → 见证不更新 |
| S-15 | 来源缺失写入拒绝 | L+P | 每条记忆必须有来源标识，缺失则返回 422 | 无来源写入 → 422 |
| S-16 | 定向遗忘未留痕视为未执行 | L+P | 定向遗忘操作写入审计日志，无日志则视为未执行 | 定向遗忘后查审计日志 → 有记录 |
| S-17 | 结构性反例免于遗忘 | L+P | P6 全局闸门守护——反例类记忆不受遗忘调度器评估，结构反例在压缩/升华管道中强制保留 | 审计日志中结构性反例无遗忘记录 |
| S-18 | Hard Delete 安全门 | L+P | 硬删除须先确认对应 vector 存储已清理完毕，才允许 SQLite/PostgreSQL 数据删除。软删除不受此限。强制执行 fail-closed：vector 清理失败则事务回滚，数据保留 | 硬删除后查 vector 存储 → 已清理；vector 清理失败 → 数据保留 |
| S-19 | 哈希净化（Hash Purification） | L+P | 定向遗忘的记忆用 SHA-256 替代原文（化石节点），保留拓扑关系但不可恢复原文。GDPR 擦除请求对应 S-19 执行 | 定向遗忘后查原文 → 被 SHA-256 替代（不可逆） |

## §2 认证与授权

### 2.1 API Key 生命周期

| 阶段 | 操作 | 适用 |
|:----|:-----|:----:|
| 生成 | `kairos admin key generate`，输出 Key + ID | L+P |
| 存储 | 环境变量 `KAIROS_API_KEY`，文件权限 600 | L+P |
| 使用 | Header `Authorization: Bearer <key>` 或 CLI 自动读取 | L+P |
| 轮换 | `kairos admin key rotate`，旧 Key 有 1 小时宽限期 | P |
| 吊销 | `kairos admin key revoke <key-id>` | P |

### 2.2 权限分级

| 角色 | 权限 | 适用 |
|:----|:-----|:----:|
| admin | 所有操作（含宪法修订/冻结/配置修改） | L+P |
| write | 记忆写入/修改/删除（不可创建或吊销其他 Key） | L+P |
| read-only | 仅检索/浏览/查询操作 | P |

## §3 数据保护

### 3.1 加密方案

| 层级 | 方案 | 用途 |
|:----|:-----|:-----|
| 数据库连接 | 环境变量 `KAIROS_DB_PASSWORD` + `KAIROS_DB_DSN` | [P] 标准/全量模式必填 |
| At-rest（DB 文件） | SQLite 加密扩展 / 全盘加密（OS 层） | 本地自用 [L] |
| At-rest（敏感字段） | AES-256-GCM，Key 来自环境变量 `KAIROS_SECRET_KEY` | [L+P] |
| In-transit（API） | HTTPS（生产环境）/ localhost 免 TLS（开发） | [P] / [L] |
| Audit HMAC Key | 独立于加密 Key，环境变量 `KAIROS_AUDIT_HMAC_KEY` | [L+P] |

### 3.2 敏感字段

标记为 `is_sensitive=true` 的字段自动加密：memory 内容、VAD 情感纹理、校准信号负载。加密在写入前、存储前执行，应用层始终处理明文。

## §4 隐私评估

| 评估项 | 本地自用 [L] | 对外开放 [P] |
|:-------|:-----------|:-------------|
| 收集什么 | 用户通过 API/CLI 写入的记忆 + 使用事件轨迹 | 同左 |
| 数据存储位置 | 用户指定路径（默认 `~/.kairos/`） | 同左 |
| 保留多久 | 由遗忘调度器自动管理 + 用户手动删除 | 同左 |
| 如何删除 | `kairos memory delete <id>` + 覆写 | 同左 |
| 用户权利（查/删/导出） | CLI `search`/`delete`/`export` 覆盖 | 同左 |
| 第三方数据共享 | 无 | 无（Embedding 模型可选本地推理） |

## §5 密钥管理

| 密钥 | 用途 | 存储位置 | 轮换周期 |
|:----|:-----|:--------|:--------:|
| `KAIROS_API_KEY` | API 鉴权 | 环境变量 / `.env` 文件 | 按需 |
| `KAIROS_SALT` | 加盐值（S-05） | 环境变量 | 季度（轮换时保留旧盐值验证已有哈希，新盐值仅用于新哈希） |
| `KAIROS_SECRET_KEY` | 敏感字段加密（AES-256-GCM，32 字节 hex） | 环境变量 | 季度 |
| `KAIROS_AUDIT_HMAC_KEY` | 审计链签名 | 环境变量 | 季度 |
| LLM Provider Key | 外部校准 | 环境变量 | 按供应商策略 |

**泄漏应急**：发现密钥泄漏 → 立即轮换该 Key → 检查审计日志确认未发生未授权访问 → 记录到 risks.md。

## §6 安全测试清单

| 测试项 | 方法 | 预期 |
|:------|:----|:-----|
| 认证绕过 | curl 不带 Key 访问所有端点 | S-01 启动级拦截：连接被拒绝。**运行时**缺少有效 Key 返回 HTTP 401。首次部署可用 `kairos init --init-key` 预注入 Key——命令自动生成全部密钥，不违反 S-01 |
| 路径注入 | `kairos search --path "../../etc/passwd"` | 返回 400 |
| 敏感字段泄露 | 直接读取 SQLite DB | 敏感字段为密文 |
| 审计链篡改 | 修改审计日志条目后校验 HMAC | 校验失败 |
| 权限提升 | read-only Key 执行写操作 | 返回 403 |
| 资源耗尽 | 并发 1000 请求 | 触发限流 429 不崩溃 |

## §7 安全事件响应

```
发现 → 评估（P0=数据泄露/P1=未授权访问/P2=配置错误）
     → 遏制（轮换密钥 / 下线服务 / 隔离数据）
     → 恢复（从备份恢复 / 回滚配置）
     → 复盘（更新 threat-model / 安全规格 / 测试清单）
```

单人开发的事件响应 = 你自己发现问题 → 你自己修 → 你自己记教训。
