# Kairos 架构风险登记册

> 本文记录当前认知层与架构层设计中已知的架构级风险与哲学张力。这些风险不阻塞 v1.0 发布，但应在 v1.x 演进过程中被持续评估与治理。

---

## 一、架构级风险

### R-001 治理/功能复杂度比偏高

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十三次独立评析（2026-07-19 83.1 分三维度评审） |
| **描述** | 宪法主权面、宪法解释层、审计庭、元认知治理器族、健康计数器、受控偏差通道、不变量修订门禁、排序例外预算、认知关节、层演化门禁、协调复杂度硬上限等元层级 machinery 的复杂度可能超过记忆操作本身的功能复杂度。v1.0.0 存在为未来可能性预付过多治理成本的风险。 |
| **影响评估** | 中。治理组件可在 v1.0 以"监控但不强制"的降级形态运行，待真实演进压力出现再升级为硬约束。 |
| **缓解建议** | 在 v1.0 实施前评估：哪些治理组件可以降级为"监控不强制"形态；哪些例外门禁可合并同类项。 |
| **跟踪版本** | v1.0 评估 → v1.1 实施 |

### R-002 元认知层职责过宽

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十三次独立评析 |
| **描述** | 检测器族（十余种）+ 治理器族 + 自观察记忆 + 健康计数器 + 受控偏差通道聚合在单一元认知层，使其成为隐性中枢。虽与宪法主权面已物理隔离，但监测与治理决策在元认知层内部未进一步分离。 |
| **影响评估** | 中。架构复杂度风险——元认知层自身故障或偏置的传播面较大。 |
| **缓解建议** | 在方向层明确元认知层的"监测"与"治理决策"是否应在更高抽象上再分。 |
| **跟踪版本** | v1.0 评估 → v1.1 或 v1.2 |

### R-003 看门人故障传播面

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十三次独立评析 |
| **描述** | 正交平面、审计庭、元认知层均为"看门人"组件。看门人越多，误冻（false freeze）或误否（false veto）的潜在面越大。当前降级路径已设计（冻结反制、降级契约），但"宁可漏判还是宁可误冻"的偏好未在方向层显式讨论。 |
| **影响评估** | 中-低。降级路径设计已覆盖基本安全场景。 |
| **缓解建议** | 方向层补充"治理组件错误偏好"讨论，明确误冻/误否的优先级选择。 |
| **跟踪版本** | v1.1 架构审查 |

### R-004 宪法主权面三权集中

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十五份深度评析（2026-07-20） |
| **描述** | 宪法主权面同时持有执行权（否决、冻结）、立法权（宪法修订端口唯一入口）和原本隶属的审计权（审计庭曾物理剥离至宪法主权面）。虽然审计庭已在 v1.0 重构中进一步剥离至独立监督平面（§1.7），但宪法主权面仍集否决/冻结执行权与宪法修订立法权于一身。在外部校准长期静默的场景下，宪法主权面成为"自我制衡"的结构——缺少独立于主权面的内部制衡力量。 |
| **影响评估** | 低-中。监督平面的独立已消除了最严重的"三权集中"风险。当前剩下的"执行+立法"两权集中，在外部校准端口存在时由外部供给制衡，仅在外部校准长期中断时成为隐性风险。 |
| **缓解建议** | 当前已缓解至可接受水平（审计庭独立 + 外部校准端口制衡）。建议在 v2.0 重新评估宪法修订端口是否需要多钥控制。 |
| **跟踪版本** | v1.0 记录 → v2.0 评估 |

### R-005 主权面强制冻结单点可用性风险

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十四份独立评析（2026-07-20，架构方向层面§3.2） |
| **描述** | 宪法主权面持有强制冻结所有内部环的至高权限（§1.2）。虽有到期自动解冻、解释层豁免、告警回执退避等缓解措施，但在弱自反性下，若冻结触发被偏置误导（误报盲区/误报解释枯竭），可能不必要地冻结全系统内部环，造成可用性降级。当前缓解措施为"事后解冻"而非"事前校验"。 |
| **影响评估** | 中。解冻机制可恢复，但误冻期间的系统不可用对实时性要求高的场景影响显著。 |
| **缓解建议** | 评估"冻结的置信度门槛"（多大比例的监测器触发才授权冻结）与"部分冻结"粒度（可否仅冻结某一层而非全部内部环）。 |
| **跟踪版本** | v1.0 评估 → v1.1 实施 |

### R-006 跨层协调协议复杂度余量偏紧

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十四份独立评析（2026-07-20，架构方向层面§3.2） |
| **描述** | 架构设跨层协调协议绝对上限 10 条 + 相对 120%，v1.0 已有 7 条，仅余 3 条增量扩展空间。社会性/具身扩展（_social/_embodied 占位）一旦激活，可能快速耗尽余量。且「新增协议须经审计庭审批」是软门禁（非强制拦截）。 |
| **影响评估** | 中-低。3 条余量在 v1.1 之前可能够用，但社会性扩展一旦启动可能需一次性新增 2-3 条协议。 |
| **缓解建议** | 在 v1.0 阶段提前设计协议合并/替代路径（如将多条原子协议合并为一条可参数化的复合协议），为未来扩展预留弹性。 |
| **跟踪版本** | v1.0 设计 → v1.1 实施 |

### R-007 宪法解释层偏置自监测缺失

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十四份独立评析（2026-07-20，设计理念层面§2.2） |
| **描述** | 宪法解释层上移主权面后，它既是抽象原则→具体情境判例的生成者，又受元优先序锁定周期管理，又通过审计庭触发复审。集中化带来单点认知风险：若解释层的价值校准出现偏差，全系统裁决随之偏差，而能纠正它的外部校准又依赖同一解释层做「解释枯竭告警」的路由。当前无独立于解释层自身的偏置监测回路——解释健康指示器监测的是解释一致性（是否自洽），而非解释方向性（是否与外部校准一致）。 |
| **影响评估** | 中。将解释层健康度纳入监督平面（§1.7）的独立监测范围，而非仅依赖自身的衰减曲线。 |
| **缓解建议** | 监督平面增设计算宪法解释层输出与外部校准信号之间的一致性/偏差率监测器。 |
| **跟踪版本** | v1.0 识别 → v1.1 纳入监督平面 |

## 二、哲学张力

### T-001 状态演化轴缺位

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十三次独立评析 |
| **描述** | 认知基础四轴全部为价值/空间维度，缺少描述"记忆从易变表征到稳定表征的时间演化流"的维度。巩固/再巩固的过程性质在认知层被代理为架构层术语。CF 已补充边界声明（§引论·巩固阶段轴边界声明），但认知模型的全过程覆盖因此被显式缩减。 |
| **影响评估** | 低。边界声明已闭合此缺口。不影响 v1.0 架构决策。 |
| **跟踪版本** | 已闭环（认知层边界声明） |

### T-002 自演化叙事的外部供给依赖

| 属性 | 内容 |
|:----|:------|
| **来源** | 多次独立评析 |
| **描述** | "自演化记忆系统"叙事与"外部校准源充分性决定系统上限"的他律性前提之间存在结构性张力。系统突破认知上限的前提是外部先告诉它边界在哪，而外部校准源未被建模为系统一等公民。 |
| **影响评估** | 高。这是体系最大的未闭合张力，已诚实标注。需在 v1.x 优先推进外部校准源充分性建模。 |
| **跟踪版本** | v1.0 已标注开放条件 → v1.1 启动建模 |

### T-003 他律性降级态下操作真理裂隙放大

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十四份独立评析（2026-07-20，设计理念层面§2.2） |
| **描述** | 系统日常以实用论（操作真理）为主导，依赖外部校准定期重置偏置。但切换条件的判定本身依赖元认知监测（触及元记忆偏置问题）。若外部校准长期静默（进入他律性降级契约的安全休眠态），系统实质上以「好用」为真理标准运行——此裂隙在降级态下被结构性放大，而非被管理。认知基础§三的「内部真理性判据的状态定义」（准真理性地位、临时性、可被后续外部校准推翻）已覆盖此风险的认知层声明，但未描述降级态下「好用」标准持续累积的纠正机制。 |
| **影响评估** | 中。外部校准恢复正常后，被临时认可的「准真理」需逐一重新校准，校准负担较大。 |
| **缓解建议** | 在他律性降级契约中显式描述「操作真理裂隙在降级态下的放大」及对应缓释机制——如在进入安全休眠态前记录最后一次外部校准的快照状态，作为降级态内部裁决的基准锚点。 |
| **跟踪版本** | v1.0 记录 → v1.1 补充降级契约 |

## 三、方法论风险

### M-001 监管过度生长

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十三次独立评析 |
| **描述** | 认知基础文档演化出庞大的元规则层（声明门禁、认知关节元判别、排序例外预算、P6 受控例外审计、统一冲突裁决框架、体系聚合可证伪性……）。当管理模型的规则复杂度接近模型本身时，存在为逻辑洁癖而非真正认知风险预付成本的倾向。 |
| **影响评估** | 中-低。当前元规则层各条均有独立正当性论证，未发现冗余。需在后续演进中持续监控。 |
| **缓解建议** | 每次新增元规则前自问："此规则是为真正的认知风险服务，还是为逻辑洁癖服务？"前者纳入，后者合并或下放。 |
| **跟踪版本** | 持续 |

### M-002 可证伪性架构承载初建

| 属性 | 内容 |
|:----|:------|
| **来源** | 第十五份深度评析（2026-07-20） |
| **描述** | v1.0 新增了耦合计监测器、VAD 独立性测试器和证伪响应机制（§8.8），将可证伪性从认知层声明首次兑现为架构层承载。但这些监测器和响应机制属于 v1.0 初建版本——耦合计监测器的阈值需实装数据校准，证伪响应流程尚未经过真实场景验证。在积累足够运行数据前，证伪信号的可靠性停留在设计级。 |
| **影响评估** | 低。初建版本的监测器以"监控不强制"形态运行不影响系统核心功能——证伪信号在 v1.0 是信息而非命令。 |
| **缓解建议** | v1.0 以数据收集模式运行（监测器记录日志、不触发告警），v1.1 根据运行数据校准阈值后正式启用。 |
| **跟踪版本** | v1.0 初建 → v1.1 校准启用 |
