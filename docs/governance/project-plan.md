# Kairos 项目计划

> **定位**：从设计冻结到 v1.0.0 可运行的里程碑分解。当前 v1.0.0 为设计冻结版本，代码尚未启动，本文为执行路线图。

---

## 一、阶段划分

```
Phase 0: 基建搭建 ───────── Phase 1: 核心存储 ───── Phase 2: 认知层 ──── Phase 3: 集成验证
  (第 1-2 周)               (第 3-6 周)              (第 7-10 周)          (第 11-12 周)
```

## 二、Phase 0：基建搭建（第 1-2 周）

| 里程碑 | 交付物 | 依赖 |
|:------|:-------|:-----|
| 开发环境就绪 | Python 虚拟环境 + Docker 环境 | 技术选型 |
| 项目骨架 | 目录结构 + CLI 入口 + 配置加载 | — |
| 数据库初始化 | PostgreSQL 表结构创建 + 迁移工具 | 数据模型 |
| CI 流水线 | 自动化测试 + lint + 类型检查 | — |

## 三、Phase 1：核心存储层（第 3-6 周）

| 里程碑 | 交付物 | 验证方法 |
|:------|:-------|:---------|
| 记忆 CRUD | `memories` 表基本操作 | 单元测试覆盖 ≥ 80% |
| 路径空间 | 路径前缀索引 + 查询 | S-04 路径隔离验证 |
| 双副本分离 | 见证锚定 + 使用权重独立更新 | S-14 语境自指禁令测试 |
| 向量检索 | pgvector 语义搜索 | 检索精度验证 |
| 关系索引 | 四类关系 CRUD | 关系查询测试 |
| 事件总线 | 事件发布/订阅/背压 | 集成测试 |

## 四、Phase 2：认知层（第 7-10 周）

| 里程碑 | 交付物 | 验证方法 |
|:------|:-------|:---------|
| 遗忘调度器 | 遗忘得分计算 + 调度 | 遗忘后记忆不可见 |
| 潜伏势能重估 | 零使用价值记忆保留机制 | 重估触发可观测 |
| 升华管道 | raw→item→strategy→behavior | 各阶段日志正确 |
| 策略层裁决器 | 帕累托 + 辞典式排序 + 真理模式 | P6 合规审计 |
| 差异检验 | 使用权重陡升 → 触发检验 | 分级触发正确 |
| 注意力调度 | 预算分配 + 背压 | 超限触发 |

## 五、Phase 3：集成验证（第 11-12 周）

| 里程碑 | 交付物 | 验证方法 |
|:------|:-------|:---------|
| 宪法主权面 | 外部校准 + 冻结 + 降级 | 状态机切换正确 |
| 元认知层 | 偏置监测 + 告警 | 偏置触发告警 |
| 监督平面 | 审计庭 + 证伪响应 | 证伪信号可路由 |
| E2E 测试 | 6 条关键路径 | 全部通过 |
| 安全红线验证 | 16/16 | 全部通过 |
| 性能基准 | 检索延迟 P50/P95/P99 | 达标 |

## 六、风险与缓冲

| 风险 | 概率 | 影响 | 缓解 |
|:----|:----:|:----:|:-----|
| pgvector 性能不达预期 | 中 | 检索延迟超限 | 准备 HNSW 索引参数调优 |
| LLM 嵌入依赖不可用 | 低 | 语义检索中断 | 本地嵌入模型 BGE-M3 备选 |
| 轻量模式 SQLite 容量瓶颈 | 低 | 5 万条上限 | 用户需切换至标准模式 |
| 跨层协调复杂度超限 | 低 | 维护成本增加 | 绝对上限 10 条协议控制 |
