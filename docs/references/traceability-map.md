# 需求可追溯性映射表

> **定位**：建立 feature-list（43 能力）、claim-matrix（37 声明）、debt-collection（11 项开放 + 7 项已闭环）、cognitive-architecture-gap（12 差距）四套分类编号之间的交叉映射。解决「哪些能力受哪些声明约束、被哪些债务延后、受哪些差距影响」的追溯断链。
>
> **映射原则**：仅建立存在直接对应关系（能力→声明→债务→差距在同一机制上）的映射，不硬造不存在的关系。空单元格表示该编号在当前其他分类中无直接对应项。
