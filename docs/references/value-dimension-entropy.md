# 多样性守护——价值维度熵值算法参考

> **认知基础映射：** 多样性原则——守护价值向量的维度多样性，非条目数量

## 一、价值维度熵值

将使用价值轴的五个特征标记（检索/验证/贡献/模拟/内隐）视为一个概率分布 `P = {p_r, p_v, p_c, p_s, p_i}`，其中：

```
p_t = count_t / total
```

- `count_t` = 类型 t 在某时间窗口内的调用次数
- `total` = 该窗口内所有类型调用总数

**熵值 H**：

```
H = -Σ(p_t × log₂(p_t))
```

- 最大熵（均匀分布）：`H_max = log₂(5) ≈ 2.32`
- 最小熵（单一类型）：`H_min = 0`

## 二、坍缩判定逻辑

| 状态 | 条件 | 动作 |
|:----|:-----|:-----|
| ✅ 健康 | H ≥ 1.0 | 无动作 |
| ⚠️ 警戒 | 0.5 ≤ H < 1.0 | 写入使用事件总线，标记 `diversity_warning` |
| 🔴 坍缩倾向 | H < 0.5 | 向策略层发出「维度收敛趋势报告」（非冻结命令）。策略层评估后决定是否触发降级——此判断不依赖单标量 H（H 仅提示趋势方向，不构成裁决），符合 P6 禁止单标量聚合裁决。外部校准静默期此报告单独不足以触发冻结，须结合其他维度指标 |

坍缩检测由 §2.2 元认知层的元记忆偏置监测器执行（`value_entropy` 子项），在每个调度周期末计算当前窗口的 H 值并比对阈值。

## 三、多窗口趋势分析

为避免短时波动触发误告警，熵值基于滑动窗口计算：

- **短窗口**：最近 10 个调度周期（用于警戒检测）
- **长窗口**：最近 50 个调度周期（用于趋势报告，仅记录不触发）
- **趋势斜率**：H 在长窗口上的线性回归斜率 < -0.05（每周期的熵值显著加速下降）时，即使 H 未低于警戒阈值也向宪法解释层提交观察报告

## 四、与使用负载算法的关系

本文的价值维度熵值检测与 `usage-load-algorithm.md` 的加权系数模型共享同一个五维使用空间，但回答不同问题：

- **本文（熵值检测）**：当前使用分布是否异常坍缩——以等概率分布为参考基线判断多样性
- **加权系数模型**：各维度对升华/遗忘优先级的相对重要性

两者不是竞争关系：熵值检测回答「当前的分布是否健康」，加权系数设定「什么重要」。同一五维向量的两种不同用途
